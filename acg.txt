===============================================================================
                             CORE CONCEPT
===============================================================================

WHAT THIS IS:
• A 90s-styled web "oracle" that generates daily ancient wisdom quips
• AI translates/creates wisdom from "untranslated" ancient texts
• Front-end shows seamless magic, backend shows all source/process data
• User-powered AI access (users provide their own AI via multiple methods)

WHAT IT IS NOT:
• Not about manually adding more ancient texts to a database
• Not just another quote generator
• Not dependent on a single API key (user-powered model)

THE VISION:
"A clean, 90s-styled HTML site where AI translation feels like magic on the 
front end, but all source material and translation data is transparently 
available in the backend. Users get their 'daily ancient wisdom thought' 
while seeing exactly how it was created."

===============================================================================
                             ARCHITECTURE
===============================================================================

FRONT-END (The Magic):
1. Single "INITIATE TRANSLATION" button - generates daily wisdom
2. Clean 90s aesthetic (Matrix green/cyber blue/retro pink)
3. Wisdom displayed as glowing, prominent text
4. No visible complexity - feels like instant oracle magic

BACK-END (The Transparency):
1. "VIEW TRANSLATION DATA" button reveals all workings
2. Shows: source material, AI process log, full JSON data, proof of work
3. All data downloadable as JSON
4. Process logs show every step

BETA PANEL (User-Powered AI):
1. Collapsible advanced panel
2. Four access methods:
   a) Public Proxy (free endpoints, default)
   b) Bookmarklet (user's own Claude/ChatGPT)
   c) Direct AI site links (Claude.ai, ChatGPT)
   d) User API key (future: connect to their own keys)
3. Users can choose/switch methods

===============================================================================
                          API INTEGRATION GUIDE
===============================================================================

SECURITY FIRST:
• NEVER put API keys in frontend JavaScript
• ALWAYS use backend proxy
• Free options: Vercel Serverless, Netlify Functions, Replit

BACKEND PROXY EXAMPLE (Node.js/Express):
```javascript
// Claude API proxy endpoint
app.post('/api/claude-translate', async (req, res) => {
  const { text, subject } = req.body;
  
  const response = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': process.env.CLAUDE_API_KEY,
      'anthropic-version': '2023-06-01'
    },
    body: JSON.stringify({
      model: 'claude-3-haiku-20240307',
      max_tokens: 500,
      messages: [{
        role: 'user',
        content: `Translate ancient text: ${text}\nSubject: ${subject}`
      }]
    })
  });
  
  const data = await response.json();
  res.json(data);
});
